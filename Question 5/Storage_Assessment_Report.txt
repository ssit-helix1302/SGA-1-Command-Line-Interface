3.

Command: df -i
Observation: I ran 'df -i' to check the inode usage across mounted file systems. The output showed the total, used, and available inodes, which is critical because running out of inodes prevents new files from being created even if physical disk space remains.

4. Storage Health Observations:
Based on the outputs, the primary FileSystem partition has adequate space available (e.g., 29% used). The inode utilization is also well within safe limits, indicating that the system is not currently at risk of storage exhaustion or metadata table overflow.
